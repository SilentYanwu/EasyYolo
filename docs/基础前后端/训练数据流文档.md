
# EasyYolo模型训练数据流全面分析

## 1. 数据流总览

| 阶段 | 前端操作 | 前端文件与函数 | 前后端通信 | 后端文件与函数 | 数据处理与存储 |
|:-----|:---------|:-------------|:-----------|:--------------|:--------------|
| 1. 数据集上传 | 用户选择并上传训练数据集zip文件 | events.js: handleDatasetUpload() | POST /upload_dataset (FormData) | training_service.py: extract_and_validate_dataset() | 保存zip到datasets目录，解压并验证data.yaml |
| 2. 训练参数配置 | 用户填写训练参数并点击"开始训练" | events.js: handleStartTraining() | POST /start_training (FormData) | training_service.py: start_training_task() | 初始化训练状态，启动后台训练线程 |
| 3. 训练执行 | 后台线程执行YOLO训练 | - | - | training_service.py: _run_yolo_training() | 加载基础模型，执行训练，保存结果 |
| 4. 训练进度监控 | 前端轮询训练进度 | events.js: startTrainingPoller() | GET /training_progress | training_service.py: get_progress() | 返回当前训练状态、进度和指标 |
| 5. 训练停止 | 用户点击停止训练按钮 | events.js: stopTraining() | POST /stop_training | training_service.py: stop_training() | 设置停止事件，终止训练线程 |
| 6. 训练结果展示 | 训练完成后展示结果图表 | events.js: loadModelDetails() | GET /training_history/{model_name} | db_service.py: get_training_record() | 从数据库获取训练记录详情 |
| 7. 队列训练模式 | 用户添加多个训练任务并启动队列 | events.js: startQueueTraining() | POST /start_training (FormData) | training_service.py: start_training_task() | 依次执行队列中的训练任务 |

## 3. 关键数据结构

### 训练状态 (training_state)
```python
training_state = {
    "model_name": None,       # 正在训练的新模型名字
    "status": "idle",         # 'idle', 'training', 'success', 'error', 'stopped', 'early_stopped'
    "progress": 0,            # 当前 Epoch
    "total": 0,               # 总 Epochs
    "metrics": {},            # 实时指标 (如 box_loss, mAP50)
    "eta": "计算中...",       # 预计剩余时间
    "error_msg": "",          # 错误信息
    "start_time": 0,          # 训练开始时间戳
    "last_epoch_time": 0,     # 上一轮结束时间戳
    "is_early_stopped": False, # 是否早停完成
    "early_stopped_epoch": 0    # 早停时的轮数
}
```

### 多任务队列状态 (multiTaskQueue)
```javascript
multiTaskQueue = {
    "tasks": [],              # 任务数组
    "currentTaskIndex": -1,   # 当前执行的任务索引
    "isQueueRunning": false,  # 队列是否正在运行
    "queueStatus": "idle"     # 队列状态: 'idle', 'running', 'stopped', 'error'
}
```

### 训练任务 (TrainingTask)
```javascript
TrainingTask = {
    "id": "",                 # 任务唯一标识
    "index": 1,               # 任务序号
    "title": "任务一",        # 任务标题
    "modelSource": "existing",# 模型来源: 'existing'或'previous'
    "baseModel": "",          # 基础模型名称
    "newModelName": "",       # 新模型名称
    "description": "",        # 任务描述
    "datasetPath": "",        # 数据集路径
    "datasetName": "未选择",   # 数据集名称
    "parameters": {},          # 训练参数
    "status": "pending",      # 任务状态: 'pending', 'running', 'completed', 'failed'
    "progress": 0,            # 当前进度
    "totalEpochs": 0,         # 总轮数
    "errorMessage": ""       # 错误信息
}
```

### 训练停止事件 (stop_training_event)
```python
stop_training_event = threading.Event()  # 用于停止训练的全局事件对象
```

### 训练记录 (TrainingHistory)
```python
class TrainingHistory(Base):
    __tablename__ = "training_history"
    id = Column(Integer, primary_key=True, index=True, autoincrement=True)
    model_name = Column(String, unique=True, nullable=False, index=True) # 训练产生的文件名 (也是key)
    base_model = Column(String)                                          # 基础模型名
    dataset = Column(String)                                             # 数据集名称
    parameters = Column(String)                                          # 训练参数 (JSON 字符串)
    description = Column(String, default="")                              # 模型介绍
    final_metrics = Column(String, default="")                            # 训练完成时最终指标 (JSON 字符串)
    is_early_stopped = Column(Integer, default=0)                       # 是否早停 (0:否, 1:是)
    early_stopped_epoch = Column(Integer, default=0)                       # 早停时的轮数
    created_at = Column(DateTime, default=datetime.now)                  # 训练完成时间
```

| 训练图表 | backend/trainchart/{model_name}/ | 训练产生的图表 |
| 训练记录 | database/yolo_app.db (TrainingHistory表) | 训练历史记录 |
| 临时训练输出 | runs/detect/{model_name}/ | 训练过程中的临时文件 |
| 多任务队列 | localStorage (multiTaskQueue) | 前端存储的训练任务队列 |

## 6. 队列训练模式特性

### 6.1 模式切换

- **单任务模式**: 传统的单任务训练模式，一次只能训练一个模型
- **多任务模式**: 支持最多6个训练任务的队列，可以依次执行
- 模式切换时会检查训练状态，防止在训练过程中切换

### 6.2 任务管理

- **添加任务**: 点击"添加任务"按钮添加新的训练任务
- **删除任务**: 可以删除未开始的任务，删除时会检查任务依赖
- **编辑任务**: 可以修改任务的参数配置
- **任务依赖**: 支持使用上一个任务的输出模型作为下一个任务的基础模型

### 6.3 队列执行

- **顺序执行**: 队列中的任务按顺序依次执行
- **自动切换**: 当前任务完成后自动开始下一个任务
- **状态跟踪**: 实时显示每个任务的执行状态（pending/running/completed/failed）
- **进度监控**: 实时显示当前任务的训练进度

### 6.4 停止机制

- **停止队列**: 可以随时停止正在运行的队列
- **停止确认**: 停止时会显示确认对话框
- **状态重置**: 停止后所有任务状态重置为pending
- **资源释放**: 停止时会释放GPU显存并清理临时文件

### 6.5 数据持久化

- **队列保存**: 任务队列自动保存到localStorage
- **状态恢复**: 页面刷新后自动恢复队列状态
- **模式记忆**: 记住上次选择的训练模式

### 6.6 错误处理

- **任务失败**: 单个任务失败时停止队列并显示错误信息
- **错误提示**: 显示详细的错误信息帮助排查问题
- **状态标记**: 失败的任务标记为failed状态

### 6.7 限制与约束

- **最大任务数**: 最多支持6个训练任务
- **运行时限制**: 队列运行时不能添加或删除任务
- **模式互斥**: 训练进行时不能切换模式

## 7. 早停机制

### 7.1 早停原理

- **触发条件**: 当验证指标（如mAP50）在连续patience个epoch内没有提升时，训练自动停止
- **patience参数**: 用户可在训练参数中设置patience值（默认20），控制早停的耐心程度
- **早停判断**: YOLO框架内部实现早停逻辑，当满足条件时自动终止训练

### 7.2 早停状态管理

- **状态标记**: training_state中添加is_early_stopped和early_stopped_epoch字段
- **数据库记录**: TrainingHistory表中添加is_early_stopped和early_stopped_epoch字段
- **状态值**: 新增'early_stopped'状态，区分正常完成和早停完成

### 7.3 早停与队列训练

- **模型可用性**: 早停的模型仍可作为后续任务的基础模型
- **队列连续性**: 队列训练中，即使任务早停，下一个任务仍可使用早停的模型
- **进度保存**: 早停时保存当前最佳模型（best.pt），确保模型质量

### 7.4 用户提示

- **完成提示**: 正常完成时显示"🎉 训练完成！新模型已就绪。"
- **早停提示**: 早停完成时显示"⚠️ 训练已早停完成！在第X轮提前结束。"
- **轮数显示**: 提示信息中包含早停时的实际轮数

### 7.5 详情页展示

- **状态标识**: 在模型详情页面显示"训练完成"或"早停完成"标识
- **轮数说明**: 如果是早停，显示"在第X轮早停完成"
- **视觉区分**: 使用不同的颜色或图标区分正常完成和早停完成
- **模型来源**: 第一个任务必须使用现有模型，后续任务可以选择使用上一个任务的输出模型 |
