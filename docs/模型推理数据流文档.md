
# EasyYolo模型推理数据流全面分析

## 1. 数据流总览

| 阶段 | 前端操作 | 前端文件与函数 | 前后端通信 | 后端文件与函数 | 数据处理与存储 |
|:-----|:---------|:-------------|:-----------|:--------------|:--------------|
| 1. 模型选择 | 用户选择要使用的模型 | events.js: handleModelSwitch() | POST /switch_model (FormData) | main.py: switch_model() | 更新全局current_model_name |
| 2. 图片上传 | 用户选择要推理的图片 | events.js: dom.imageInput.onchange | - | - | 保存到selectedFiles变量 |
| 3. 单张图片推理 | 用户点击开始推理按钮 | events.js: handleSinglePredict() | POST /predict (FormData) | inference_service.py: handle_single_predict() | 保存图片，执行推理，存入数据库 |
| 4. 批量图片推理 | 用户选择多张图片并点击推理 | events.js: handleBatchPredict() | POST /predict_batch (FormData) | inference_service.py: batch_predict_generator() | 逐张推理并推送进度(SSE) |
| 5. 视频推理 | 用户选择视频并点击推理 | events.js: handleVideoPredict() | POST /predict_video (FormData) | inference_service.py: video_predict_generator() | 逐帧推理并推送进度(SSE) |
| 6. 结果展示 | 推理完成后展示结果 | events.js: ui.renderHistory() | - | - | 从数据库获取历史记录并展示 |

## 3. 关键数据结构

### 推理历史记录 (InferenceHistory)
```python
class InferenceHistory(Base):
    __tablename__ = "inference_history"
    id = Column(Integer, primary_key=True, index=True, autoincrement=True)
    model_name = Column(String, nullable=False, index=True)
    original_url = Column(String)
    result_url = Column(String)
    created_at = Column(DateTime, default=datetime.now)
```

### 推理结果数据
```python
{
    "result_url": "http://xxx/inferecord/results/xxx.jpg",
    "original_url": "http://xxx/inferecord/uploads/xxx.jpg",
    "time": "2023-xx-xx xx:xx:xx"
}
```

## 4. 关键API端点

| 端点 | 方法 | 功能 | 参数 | 返回值 |
|:-----|:-----|:-----|:-----|:-------|
| /switch_model | POST | 切换当前模型 | model_name, category (FormData) | {"status": "success", "current_model": "..."} |
| /predict | POST | 单张图片推理 | file (FormData) | {"result_url": "..."} |
| /predict_batch | POST | 批量图片推理 | files (FormData) | SSE流 |
| /predict_video | POST | 视频推理 | file (FormData) | SSE流 |
| /history | GET | 获取历史记录 | model_name (查询参数) | 推理记录列表 |
| /history/{record_id} | DELETE | 删除单条记录 | record_id (路径参数) | {"status": "success"} |
| /history | DELETE | 清空历史记录 | model_name (查询参数) | {"status": "all cleared"} |

## 5. 数据存储位置

| 数据类型 | 存储位置 | 说明 |
