
# EasyYolo模型管理数据流全面分析

## 1. 数据流总览

| 阶段 | 前端操作 | 前端文件与函数 | 前后端通信 | 后端文件与函数 | 数据处理与存储 |
|:-----|:---------|:-------------|:-----------|:--------------|:--------------|
| 1. 模型列表获取 | 页面加载时获取所有可用模型 | events.js: refreshApp() | GET /models | main.py: get_models() | 扫描模型目录并返回模型列表 |
| 2. 模型上传 | 用户上传自定义模型 | events.js: handleModelUpload() | POST /upload_model (FormData) | main.py: upload_model() | 保存模型到models/yolo目录 |
| 3. 模型切换 | 用户切换当前使用的模型 | events.js: handleModelSwitch() | POST /switch_model (FormData) | main.py: switch_model() | 更新全局current_model_name |
| 4. 模型重命名 | 用户重命名模型 | events.js: handleModelRename() | POST /rename_model (JSON) | main.py: rename_model() | 重命名模型文件并更新数据库 |
| 5. 模型删除 | 用户删除模型 | events.js: handleModelDelete() | DELETE /delete_model | main.py: delete_model() | 删除模型文件和相关记录 |
| 6. 模型描述更新 | 用户更新模型介绍 | events.js: confirmEditDescription() | POST /update_model_description (JSON) | main.py: update_model_description() | 更新数据库中的模型描述 |

## 3. 关键数据结构

### 模型列表数据
```python
{
    "models": {
        "raw": ["yolo11n.pt", "yolo11s.pt", ...],
        "yolo": ["custom1.pt", "custom2.pt", ...],
        "trained": ["trained1.pt", "trained2.pt", ...]
    },
    "current_model": "yolo11n.pt"
}
```

### 模型详情数据
```python
{
    "model_name": "trained1.pt",
    "base_model": "yolo11n.pt",
    "dataset": "NEU-DET",
    "parameters": "{"epochs": 50, "batch": 16, ...}",
    "description": "这是一个用于钢材表面缺陷检测的模型",
    "time": "2023-xx-xx xx:xx:xx"
}
```

## 4. 关键API端点

| 端点 | 方法 | 功能 | 参数 | 返回值 |
|:-----|:-----|:-----|:-----|:-------|
| /models | GET | 获取所有可用模型 | 无 | {"models": {...}, "current_model": "..."} |
| /upload_model | POST | 上传自定义模型 | file, custom_name (FormData) | {"status": "success", "filename": "..."} |
| /switch_model | POST | 切换当前模型 | model_name, category (FormData) | {"status": "success", "current_model": "..."} |
| /rename_model | POST | 重命名模型 | old_name, new_name, category (JSON) | {"status": "success", "new_name": "..."} |
| /delete_model | DELETE | 删除模型 | model_name, category (查询参数) | {"status": "success"} |
| /update_model_description | POST | 更新模型描述 | model_name, description (JSON) | {"status": "success"} |

## 5. 数据存储位置

| 数据类型 | 存储位置 | 说明 |
|:---------|:---------|:-----|
| 原始模型 | models/raw/ | 官方预训练模型，不允许重命名和删除 |
| 自定义模型 | models/yolo/ | 用户上传的自定义模型，允许重命名和删除 |
| 训练模型 | models/trained/ | 用户训练的模型，允许重命名和删除 |
| 训练图表 | backend/trainchart/{model_name}/ | 每个模型的训练图表 |
| 推理记录 | database/yolo_app.db (InferenceHistory表) | 推理历史记录 |
| 训练记录 | database/yolo_app.db (TrainingHistory表) | 训练历史记录 |
