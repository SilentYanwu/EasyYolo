# 🚀 EasyYolo 开发日志

记录 EasyYolo 从零开始的成长点滴。

---
## 📅 2026/04/25
- **早停机制 (Early Stopping Mechanism)**：
    - **早停检测**：基于patience参数实现训练早停，当验证指标连续N轮无提升时自动停止训练。
    - **状态标记**：在数据库中记录模型是否早停及早停时的轮数信息。
    - **队列兼容**：早停的模型仍可作为后续任务的基础模型，确保队列训练的连续性。
    - **提示优化**：早停完成时显示明确的提示信息，包含早停轮数。
    - **详情展示**：在模型详情页面新增早停状态显示，区分正常完成和早停完成。

## 📅 2026/04/22
- **队列训练系统 (Queue Training System)**：
    - **多任务模式**：新增队列训练模式，支持最多6个（可以修改）训练任务的顺序执行。
    - **任务管理**：实现了任务的添加、删除、编辑功能，支持任务间的依赖关系。
    - **模型传递**：支持使用上一个任务的输出模型作为下一个任务的基础模型，实现模型迭代优化。
    - **状态持久化**：任务队列自动保存到localStorage，页面刷新后自动恢复。
    - **进度监控**：实时显示每个任务的执行状态和训练进度。
    - **错误处理**：单个任务失败时停止队列并显示详细错误信息。

## 📅 2026/04/21
- **停止训练功能**：
    - **停止按钮**：在模型训练界面增加停止按钮，支持在训练过程中随时停止训练。
    - **停止机制**：使用threading.Event()实现优雅的训练停止，确保资源正确释放。

## 📅 2026/04/19
- **Vue3 前端重构 1 (Vue3 Frontend Refactoring 1)**：
    - **框架迁移**：初步完成了从原生JS到Vue3+TypeScript+Element Plus的前端框架迁移，提升了代码可维护性和开发效率。
    - **代码结构优化**：采用组合式API(Composition API)重构了应用逻辑，实现了响应式状态管理和更清晰的代码组织。

## 📅 2026/04/09
- **模型详情优化 (Model Details Optimization)**:
    - **图表展示增强**：修复了模型详情界面中训练结果图片展示不全的问题。
    - **兼容性提升**：新增对 YOLO11 生成的新型图表（如 BoxF1, BoxPR, BoxP, BoxR 曲线）的自动识别与展示支持。

## 📅 2026/03/18
- **故障修复与体验增强 (Bug Fixes & UX)**：
    - **描述显示修复**：修复了模型介绍无法出现的问题。
    - **重命名防丢失机制**：增加了模型重命名后的追踪功能。解决了模型重命名后，以其为基础模型训练的子模型在详情界面丢失基础模型超链接的问题，现在可以实现全路径追踪。
    - **侧边栏快捷管理**：在侧边栏模型列表（含识别页与详情页）增加了悬浮触发的三个实心圆（⋮）快捷菜单，集成**重命名**、**修改介绍**、**删除模型**功能。

## 📅 2026/03/17
- **YOLO 训练与详情构建**：
    - **训练功能完备**：完成了第二部分模型训练的主要功能，支持模型训练，参数调整，在每轮训练结束后，模型评估结果会实时显示。
    - **详情界面搭建**：完成了第三部分模型详情界面的大部分功能，包括模型训练参数显示，模型训练历史展示。

## 📅 2026/03/16
- **视觉风格统一 (UI/UX Refinement)**：
    - **按钮组件重构**：引入了 `.btn-action` 通用类，将“播放/暂停”、“下载结果”与“清空历史”按钮统一为药丸型 (Pill) 设计。
    - **模态框统一和优化**：对导入模型，删除模型，删除数据和抛出警告等模态框进行了统一风格调整。

## 📅 2026/03/15
- **前端架构模块化 (Refactoring)**：
    - 彻底告别单体脚本，将前端逻辑拆分为：`api` (后端交互)、`ui` (渲染控制)、`events` (事件绑定)、`dom` (元素映射)、`state` (全局状态) 等 ES6 模块。
    - 提升了代码的可维护性，降低了不同功能模块间的耦合度。
- **全链路视频识别支持**：
    - **视频上传与流式推理**：支持 `.mp4` 视频导入，结合 **OpenCV 帧处理** 与 **SSE 技术** 实现推理进度实时反馈（显示当前帧/总帧数百分比）。
    - **Web 原生播放**：通过 HTML5 Video 标签实现识别结果的在线即时预览，无需下载即可观看处理后的标注视频。

## 📅 2026/03/11
- **资源闭环管理**：
    - 实现了 **“库-盘同步删除”** 机制：删除识别记录时自动物理擦除硬盘上的原始图与结果图，彻底解决冗余积累问题。
    - 优化了数据库存储逻辑，支持本地时区时间戳，更符合人类阅读习惯。
- **视觉感知优化**：
    - **高级感占位符**：由 SVG 图标与动态 CSS 构建的占位 UI 替代了传统的“破裂图片”，提升了无图状态下的视觉美感。
    - **交互微创新**：引入悬浮触发的单条历史删除按钮，支持丝滑的局部刷新展示。

## 📅 2026/03/10
- **批量识别 (Batch Inference)**：
    - 借鉴 **生产者-消费者** 队列思路，实现多图并发处理。
    - 引入 **SSE (Server-Sent Events)** 技术，前端实时渲染推理进度条与中间结果。
- **架构重构与解耦**：
    - 将所有推理相关逻辑（保存、推理、存库）封装至 `inference_service.py`。
    - 实现了单张与批量推理逻辑的高度复用，大幅简化了 `main.py` 的接口实现。
- **模型管理增强**：完善了导入模型的管理逻辑，支持更灵活的切换与配置。

## 📅 2026/03/09
- **侧边栏改进**：增加了侧边栏折叠/隐藏按钮，扩大了图片展示区域。
- **视觉迭代**：优化了整体 CSS 样式，使界面更加现代与精致。

## 📅 2026/03/08
- **核心框架**：完成了基于 FastAPI (后端) 和 原生 JS (前端) 的基础架构搭建。
- **模型导入**：初步实现 `.pt` 模型文件的上传与目录管理功能。
- **持久化方案**：
    - 引入 SQLite 存储识别历史记录。
    - **独立记录逻辑**：每个模型拥有专属的历史记录，互不干扰。
